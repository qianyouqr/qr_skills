# Playbook 00 — 研究工具 / 数据获取 / 配置（Meta）

> 触发：用户问的是**工具本身**，不是数据。例如：怎么接入飞书 / 微信、API 配置、skill 下载、报告模板、提示词、长会话、模型 ID、表格显示规则、为什么两个 agent 回答不一样。

## 1. 这一类**不调** quant-buddy

本类问题与行情数据无关。回答原则：

1. 只回答**确认能回答**的部分（agent 自己知道的、文档里写的）；
2. 不知道的部分**直说不知道**，给出指向（官方文档 / 帮助中心链接，必须是真实存在的链接）；
3. **禁止杜撰**功能特性、配置项、API 路径、产品 roadmap。

## 2. 常见子场景

| 子场景 | 处理 |
|--------|------|
| "如何接入飞书 / 企业微信 / Clawbot" | 给步骤 + 指向官方接入文档；不要凭印象编命令 |
| "XX skill 在哪下载" | 如果 agent 不知道精确链接，直说"该 skill 下载入口我无法确认，建议在 OpenClaw / ArkClaw 控制台搜索" |
| "给出股票深度分析提示词 / 模板" | 可直接产出（参考 03 playbook 的输出骨架） |
| "为什么 hermes 安装 qveris 没有你的分析详细" | 这是产品对比题——只能基于已知事实回答能力差异，不要替对方厂商背书 |
| "怎么实现表格扩列满宽 / 横向滚动" | 前端样式问题，给 CSS / Markdown 渲染建议 |
| "模型 ID 是什么" | 直接回答当前 agent 使用的模型；不知道就说不知道 |
| "如何读飞书卡片" | 是平台 SDK 问题，给已知 SDK 用法或指向官方文档 |

## 3. 输出骨架

```
【一句话答】<最直接的答案>
【步骤 / 配置】（如适用，分点）
【可参考链接】（必须真实，不要瞎编 URL）
【可能的限制 / 已知 issue】（如适用）
```

## 4. 失败兜底特化

- 用户问的工具特性 agent **完全不了解**：直接 §6，明确"对该功能的实现细节我无法确认"；不要用"通常这类工具会..."硬撑。
- 用户问"为什么 agent A 和 agent B 答案不一样"：如实说"两边使用的模型 / skill 版本 / 数据源可能不同，无法在我这一侧验证 agent B 的具体配置"。
