# Playbook 06 — 基金 / ETF / 可转债 / 指数产品

> 触发：6 位基金代码、"联接 / LOF / 场内 / 场外 / PE 分位 / 持仓占比 / 可转债 / 指数增强 / 调仓换基"。

## 1. 资产识别

- ETF（场内）：6 位代码以 5 / 1 开头，如 513310、159995、516020、588200。grep `assets_db/index.yaml` 或专门的 ETF 列表；不在表中 → quant-buddy 仍可能直接 fast_query 拿到。
- 联接 / 场外基金：6 位代码如 000834、016873、017653。**quant-buddy 对场外基金的覆盖较弱**：先 `confirmDataMulti` 验证字段，能拿到 → 走；拿不到 → §6 明确告知。
- 可转债：6 位代码（11/12 开头）；quant-buddy 是否覆盖 → 先 `confirmDataMulti` 验证；不覆盖 → §6。

## 2. 必跑数据清单

| # | 数据 | 入口 |
|---|------|-----|
| 1 | 净值 / 现价 / 涨跌幅 / 规模 / 折溢价（场内） | `fast-snapshot` |
| 2 | 近 1 月 / 3 月 / 1 年 / 3 年 累计收益 | `period-return-compare` |
| 3 | **跟踪指数的 PE / PB 分位**（场内 ETF / 指数基金的核心问题） | `quant-standard`：构造 percentile 公式（如近 5 年百分位） |
| 4 | 持仓占比（前十大持仓） | `confirmDataMulti` 先验证字段 → `fast_query` 取；不存在 → 联网取一手季报 |
| 5 | 资金流入 / 份额变化（用户问"主力资金变动"时） | `confirmDataMulti` 先验证 |

> **PE 分位** 是高频需求（恒生科技 PE 分位、513040 PE 分位…）：必须明确**分位的窗口长度**（默认建议 5 年），并在输出里写出口径。

## 3. 输出骨架

```
【结论行】<基金/ETF>(代码) 当前 <价格/净值>，跟踪 <指数>，PE 分位 X%（近 5 年）。
【核心数据】现价 / 净值 / 规模 / 折溢价 / 跟踪误差
【收益对比】近 1 月 / 3 月 / 1 年 / 3 年 vs 标的指数 vs 同类
【估值定位】跟踪指数的 PE / PB / 股息率 + 历史分位（口径说明）
【持仓 / 行业分布】前十大或行业分布（来源标注）
【建议】超买 / 合理 / 低估 + 仓位建议
【风险】+【声明】
```

## 4. 特殊场景

- **"按板块推荐 5 只补涨龙基金"**：先要用户明确板块；用 `quant-standard` 在 ETF 池里按"近 X 月跑输 + 近 1 周转强"双条件筛选；找不到候选 → §6。
- **"调仓换基建议"**：用户必须提供现持仓和目标偏好（红利 / 成长 / 主题）；缺一项 → 反问；仍不给 → §6。
- **"可转债投资价值"**：必须包含转股价值、纯债价值、转股溢价率、剩余年限；任一字段 quant-buddy 不可得 → 联网到一手转债公告；都没有 → §6。
- **"中证 500 指增 vs A500 哪个更优"**：用 `period-return-compare` 跑两个指数的同期对比 + 给出年化 / 最大回撤；定性论证用 agent 推理但**不替代数据**。

## 5. 失败兜底特化

- 场外基金 quant-buddy 不可得：明确"该场外基金 quant-buddy 数据不可得，建议提供天天基金 / 蚂蚁财富的最新净值与持仓页"——不要用任何"约 / 大致"的数字。
- "持仓占比"歧义：是基金前十大持仓 vs 该 ETF 在某行业占比 vs 公募基金对该股的持有？**反问后再继续**。
