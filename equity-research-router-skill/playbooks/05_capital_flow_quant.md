# Playbook 05 — 资金面交易策略与量化选股

> 触发：主力 / 机构 / 散户 / 大单 / 资金流 / 换手率 / 涨停 / 拉板 / FVG / SMC / 通达信公式 / 选股 / 涨停概率 / 今日看涨。

## 1. 数据可用性自检（先做，决定能否继续）

调用 `confirmDataMulti` 验证以下字段在 quant-buddy 中是否存在 + 维度对齐：
- 主力净流入 / 大单净流入 / 散户净流入（日级）
- 换手率（日级）
- 涨停 / 跌停标记
- 分钟级行情（如用户问"今天上午成交量"）

**任一关键字段不可得**：直接 §6，明确告诉用户"该资金流字段不可得"，不要拿"成交额"伪装成"主力净流入"。

## 2. 必跑数据清单（按子场景）

### 2A 单股资金流（"分析 XX 今日成交 / 主力 / 散户"）
1. 当日 / 近 5 日资金流构成（主力 / 大单 / 中单 / 小单）；
2. 当日成交额 / 换手率 / 振幅；
3. （可选）分钟级量价（quant-buddy 若有则用，注意 `refreshSnapshotTime`）。

### 2B 选股（"选出今日符合 XX 策略的股票"、"找出 API 中转概念股"、"列出 10 只能拉板"）
1. **必须先把"策略"翻译成 quant-buddy 的公式语言**（看 `quant-standard.md` + `recipes/*.md`）。
2. 不能直接翻译的（FVG / SMC / 缠论等纯技术形态）：先用 quant-buddy 算子能近似的版本（如跳空缺口可用前一日最高 < 当日最低），并**明确告知用户这是近似口径**；完全无法表达 → §6。
3. 全市场跑 `runMultiFormulaBatchStream` + TopN，得到候选；对每只候选取行情快照展开。

### 2C 通达信 / 公式工具类（"根据 XX 写通达信公式"）
- 这类**不需要跑数据**；agent 直接产出通达信语法的公式文本。
- 但**必须**告诉用户：通达信公式由 agent 生成，准确性由用户在通达信软件中自行回测验证；本工具不为该公式跑回测（回测可改走 quant-standard）。

### 2D 涨停 / 涨停概率（"明日涨停概率"、"光华股份涨停原因"）
- "原因"：用 quant-buddy 取当日资金流 + 板块涨幅 + 用 byted-web-search 取消息面，三点合一；
- "概率 / 短期高点"：**没有公认的量化模型可直接给概率**——必须明确"基于最近 N 日涨停频次 + 当前位置"的统计描述，并标"非确定性预测"；用户坚持要点估计 → §6。

## 3. 输出骨架

```
【结论行】<标的> 今日资金面 <方向>，主力 X.X 亿（净流入/出），换手 X%。
【资金流构成】主力/大单/中单/小单 净流入（标数据时点）
【量价配合】成交额、换手率、振幅、是否放量
【板块联动】所属板块当日表现 + 强度排名
【消息面】（byted-web-search 搜索，标来源 + 时点）
【信号判定】策略命中 / 未命中 + 触发条件
【风险】+【声明】
```

## 4. 特殊场景

- **"列出 10 只能拉板的股票，找出持有这些股票的基金代码"**：
  - 第一步（选股）：明确"能拉板"的可量化口径——用 `近 N 日涨停次数 ≥ X` 或 `近 5 日强度排名 + 当前未触板`，而不是凭感觉；
  - 第二步（基金持仓）：quant-buddy 是否提供"基金持有某股"反向查询？先 `confirmDataMulti` 验证；不可得 → 明确说"基金持仓反查 quant-buddy 不可得，需联网到天天基金"；联网也不全 → §6 部分失败。
- **"从资金流向分析一下"**（无主语）：反问"分析哪只 / 哪个板块 / 大盘整体"，否则 §6。
- **"博杰股份今日是否看涨"**：跑 2A 流程；输出基于已有数据的方向判断 + 概率描述，不能给"必涨 / 必跌"。

## 5. 失败兜底特化

- 资金流字段 quant-buddy 当天不可得（盘中尚未结算）：必须明示数据时点和限制，建议盘后再看；不要用 09:30 的截面伪装成全天结果。
- 港 / 美股资金流：A 股口径的"主力 / 散户"在港 / 美股**通常不可得**——直接告知不可得，仅给可得的成交量 / 换手率。
