# Playbook 07 — 港股 / 美股 / 海外标的

> 触发：港股代码（4–5 位 / .HK）、美股代码（.O/.N/.A 或英文 ticker）、英文公司名（Tesla / Intel / NVDA / TSM / ARM / MU / SanDisk / AMD…）。

## 1. 资产识别（坑特别多）

- 港股：`grep` `assets_db/stock_hk.yaml`。注意"腾讯 700.HK / 00700.HK"两种写法都要尝试；"中芯国际"既有 A 股 688981 又有港股 00981.HK，**必须问清是哪个市场**。
- 美股：`grep` `assets_db/stock_us.yaml`，关注后缀 `.N`（NYSE）/ `.O`（NASDAQ）/ `.A`（AMEX）。中文俗称（"闪迪 → SNDK.O"、"美光 → MU.O"、"台积电 → TSM.N"、"特斯拉 → TSLA.O"、"英特尔 → INTC.O"、"豪威集团" 实为 A 股韦尔股份的子公司，**容易误认**——必须先澄清）。
- "未上市但热门" 的（DeepSeek、MiniMax、SpaceX、字节跳动）：quant-buddy 不覆盖 → 走 02 行业 playbook 的"未上市公司"分支。

## 2. 必跑数据清单

| # | 数据 | 入口 | 备注 |

| 5 | K 线 | `render-kline` | — |

## 3. 非数据信息

- 美股财报电话会议要点（必须有可引用的一手来源）；
- SEC 10-K / 10-Q（如用户深度分析要求）；
- 港交所披露易公告；
- 行业地位 / 竞争对手。

## 4. 输出骨架

同 03 个股深度，但额外加：

```
【市场 / 后缀确认行】<标的>: 00700.HK（港股，港币计价）/ TSLA.O（美股，美元计价）
... 之后正文
```

汇率口径：所有数字保持原币种，不要"自动换算成人民币"——除非用户明确要换算且给定汇率口径。

## 5. 特殊场景

- **"港股代码 6082 现在价格 49.5 投资建议"**：
  - 6082 港股 5 位规范化为 06082.HK；先 grep 资产库，**如果根本没有 06082**（壁仞科技 2026 年 5 月时点未必上市）→ §6，明确告知"该代码未在港股资产库中找到，可能未上市或拼写有误"。
- **"中芯国际 00981.HK"**：必须区别于 A 股 688981.SH，输出时明确标注是港股版本。
- **"AMD 财报里的供应链变化"**：是定性解读，需要联网取最新一期 10-Q 关键披露；无来源不写。
- **"MiniMax 智谱 40 / 800 时点纳入恒科指数"**：是指数纳入规则问题，非行情；联网到一手指数公告或恒指官网；找不到 → §6。

## 6. 失败兜底特化

- 港 / 美股财务字段在 quant-buddy 返回 N/A：明示"该字段对该标的不可得"，仅给行情结论，**不得**用 A 股口径硬套或外网估算。
- 时区：美股盘中数据可能比国内有延迟；明确"数据时点（北京时间 / 美东时间）"。
