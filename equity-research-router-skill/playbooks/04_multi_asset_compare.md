# Playbook 04 — 多标的对比与组合配置

> 触发：≥2 个标的（≤10 只）+ "对比 / 比较 / 帮我分析这几只 / 对标 / 推荐 N 只 / 组合 / 仓位 / 调仓 / 买入时机"。
> **唯一数据入口：对每只标的各调用 1 次 `stockProfile`**。本 playbook 全程不调用 `fast_query` / `fast-snapshot` / `fast-window` / `fast-report-period` / `period-return-compare` / `quant-standard` 等任何其他工具（"推荐 N 只对标"的初始筛选例外，见 §4）。
> **标的数量限制**：≤10 只走本 playbook；>10 只告知用户超出上限，引导改用量化选股路由，不继续执行。

---

## 1. 资产识别

- 把用户给的所有代码 / 名称都跑一遍 grep `assets_db/*.yaml`，得到 ticker 列表。
- A / H 同名默认取 A 股（见 SKILL.md §1 第 5 条）；港 / 美股有真实歧义时先问一句。
- 标的数 >10：告知用户"本路由支持最多 10 只标的横向对比，超出部分请缩减或改用量化筛选功能"，不继续执行。

---

## 2. 唯一数据调用

### 2.0 输出合同锁定（必须先于取数）

在调用任何一只标的的 `stockProfile` 前，先锁定本轮最终答案版式：

1. **必须使用 §4 的横向 Markdown 表格骨架**：行情估值、财务、资金、计算维度、波动率都必须是表格。
2. **必须保留独立章节**：`## 五、计算维度横向对比` 与 `## 六、波动率横向对比` 分开输出；不得合并成“波动率与趋势结构 / 技术面与波动率”等标题。
3. **必须先建立填槽骨架**：在写正文前，先固定 `标题 → 时间/来源 → 一～七大节 → 横向表格表头 → 计算维度横向容器 → 结尾声明`；后续只允许把各标的 `stockProfile` 结果填入槽位，不允许临时改写成聊天摘要或项目符号。
4. **必须先计划计算维度分组横向表**：取到各标的 `stockProfile` 后，先枚举每只标的 `data.dimensions` 中未被常规模板覆盖的维度；每个维度单独输出小标题和横向指标表。
5. **群聊只压缩点评，不压缩结构**：飞书群聊可以减少解释句，但不能删除表格、不能改成项目符号、不能删 `---` 分节线。
6. **禁止过程话术外露**：首句必须直接是对比标题。禁止在标题前写"我先把 X 拿到手 / 我先按 PowerShell 规范 / 资产命中 SH600010 / 数据已拿到 / 分桶完毕 / 现在按 playbook 04 §4 输出 / byted-web-search skill 未安装 / 先识别 shell / 路径不存在"等。完整黑名单见 `SKILL.md` §1 规则 14。
7. **群聊默认走计算维度横向简洁模式**，单聊或显式"详细/完整/全部"走详细模式（沿用 `03_single_stock_deep_dive.md` §3.5 的 TopN 规则，N=3；多标的横向表每个维度 N 列）。

**一次写对的生成顺序（强制）**：
1. 先把 §4 输出骨架作为最终答案模板，不写自由文本草稿。
2. 对每只标的调用 `stockProfile` 后，先做全量维度并集：常规模板维度进入固定横向表，其余非空维度进入 `## 五、计算维度横向对比`。
3. 为 `## 五、计算维度横向对比` 先按维度并集生成所有 `### {维度名}横向对比` 小标题，再在每个小标题下生成该维度自己的横向指标表。
4. 最后才写对比结论和配置观察；点评不得改变骨架、不得合并章节。
5. 若某标的缺少某维度或指标，对应单元格填 `⚫`；不得因为缺字段把横向表改成纯文字说明。

**对列表中每只标的依次调用 1 次 `stockProfile`**（参数格式同 03 playbook §2）：

```json
{
  "asset": "<标的代码或名称>",
  "user_query": "<用户原始问题>"
}
```

| 返回情况 | 处理 |
|---------|------|
| 全部 `code:0` 且 `indicators_count > 0` | 进入 §3 输出 |
| 某只失败（< 半数） | 该标的所有表格字段填 ⚫，不以其他工具补位，其余正常输出 |
| 超过半数标的失败 | §5 整体失败 |

**不存在任何补位调用**：禁止 fast_query / period-return-compare / 任何其他 quant-buddy workflow。

## 4. 输出骨架

> **§7 格式门禁全部保留：中文大节 + `---` + Markdown 表格 + emoji 信号色 + 结尾声明行。**
> **横向表每只标的一行，缺失字段填 ⚫，禁止纯文字列表替代表格。**
> **动态维度强制覆盖**：对每只标的先枚举 `data.dimensions` 的全部非空维度；常规模板展示后，任何未展示维度 / 未展示指标都必须进入“计算维度横向对比”。不得因为维度名不在本 playbook 中就跳过。
> **计算维度必须分组**：禁止把多个计算维度混在一张大表里。每个计算维度必须有自己的 `### {维度名}横向对比` 小标题，小标题下放该维度自己的横向指标表。
|> **计算维度展示模式**：群聊默认 Top3 简洁模式（沿用 03 §3.5 选维规则），单聊或显式详细展开全量。横向表每个维度 N 列、最多 5 行指标。
> **禁止合并章节**：`## 五、计算维度横向对比` 和 `## 六、波动率横向对比` 必须分开；不得输出“波动率与趋势结构 / 技术面与波动率”等合并标题。
> **禁止压缩成列表**：下方所有表格必须保留 Markdown 表格形态；不得改写为项目符号或冒号列表。

## 4. 特殊场景

- **"推荐 N 只对标 XX"**：先确定"对标"维度（同行业 / 同 PE 区间 / 同市值量级），不明确就反问一句；明确后用 `quant-standard` 跑筛选 + TopN 拿候选列表 → 再对候选列表循环调用 `stockProfile` 出对比表。**筛选阶段调用 `quant-standard` 是唯一例外**，对比展示阶段仍只用 `stockProfile`。
- **"整理群里本周提到的所有 A 股"**：用户必须提供股票列表，否则 §6。
- **"调仓换基"**：用户必须给现持仓 + 目标，否则 §6。
- **用户明确要求某标的的 MA / KDJ / 主力净流入 / 精确支撑压力位**：§6；原因同 03 playbook §5。

---

## 5. 失败兜底特化

- **部分标的失败（< 半数）**：在表格对应行所有字段填 ⚫，备注"stockProfile 未返回该资产数据"，其余标的正常输出。
- **超过半数标的失败**：§6 整体失败；不输出残缺对比表。
- **用户给的代码在资产库中不存在 / 拼错**：先 grep 模糊匹配列出候选让用户确认；用户不应答 → §6。
- **标的数 >10**：告知超出本路由上限（10 只），不执行；引导用户缩减或改用量化选股路由。
